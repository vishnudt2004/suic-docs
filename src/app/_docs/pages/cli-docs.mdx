# Simple UI Components – CLI Documentation

(**suic-cli**)

---

## Introduction

`suic-cli` is a command-line tool to manage **Simple UI Components** in your project. It allows you to initialize **suic** in your project, add, remove, and list components easily.

**Important:** Before using `add`, `remove`, or `list` commands, you **must** run `suic-cli init`.

- **Inspired By:** The installation pattern in this project is inspired by [**shadcn/ui**](https://ui.shadcn.com/docs/cli).
- **Heads Up:** `suic-core` isn’t published as a package by design. It’s a personal collection of components. You can fork or clone the [repo](https://github.com/vishnudt2004/suic-core) from GitHub if you want to use it directly.

## Installation

Use `npx`, `pnpx dlx`, or `yarn dlx` to run the CLI:

```bash
# npx
npx suic-cli init

# pnpm
pnpm dlx suic-cli init

# yarn
yarn dlx suic-cli init
```

<Callout title="Tip">

It’s recommended to run the CLI with `npx`, `pnpm dlx`, or `yarn dlx` instead of installing it globally.

</Callout>

## Usage

```plaintext
Usage: suic-cli [options] [command]

Commands:
  init                    Set up Simple UI Components in your project
  add [components...]     Add one or more components to your project
  remove [components...]  Remove one or more components from your project
  list                    Show all available components with descriptions
  help [command]          Display help for command

Options:
  -V, --version           output the version number
  -h, --help              display help for command
```

## Commands

### Init (mandatory)

Initialize your project to use `suic-cli`. This creates the config file, sets up the installation directory, and installs the global internal dependencies.

- add path alias automatically in `tsconfig.json` (./suic/\*)

**Syntax:**

```bash
suic-cli init [options]
```

**Options:**

```plaintext
-i, --install-path <path>  Custom installation directory (default: src/suic)
```

**Example:**

```bash
suic-cli init
suic-cli init -i src/ui
```

<Callout type="warning" title="Warning">

- Running `init` again **overrides modified files**.
- Avoid manual edits to **`suic.config.json`**. Always re-run `init` to set or change paths.

</Callout>

---

### Add Components

Add one or more components to your project.

- If no components are specified, CLI prompts selection from registry.
- Names are **case-insensitive** (`button` = `Button`).

**Syntax:**

```bash
suic-cli add [components...]
```

**Examples:**

```bash
# Add a single component
suic-cli add Button

# Add multiple components
suic-cli add Button Card Carousel
```

<Callout type="warning" title="Warning">

Do not touch **`.suic/installed-registry`** — managed automatically by SUIC.

</Callout>

---

### Remove Components

Remove one or more installed components from your project.

- If no components are specified, CLI lists installed ones for selection.
- Only components from the registry are removed.
- Component files and empty dirs cleaned automatically.

**Syntax:**

```bash
suic-cli remove [components...]
```

**Example:**

```bash
suic-cli remove Button Card
```

<Callout type="warning" title="Warning">

- CLI logs dependencies/devDependencies/peerDependencies no longer required.
- Uninstall such dependencies **only if not used elsewhere in your project**.

</Callout>

---

### List Components

Display all available components in the registry, along with their descriptions and documentation URLs.

**Syntax:**

```bash
suic-cli list
```

**Example Output:**

```plaintext
Available components:
═ ═ ═ ═ ═ ═ ═ ═ ═ ═ ═

 ● Button:
   A clickable button element
   Docs: <doc-url>

 ● Card:
   A container for content with shadow and padding
   Docs: <doc-url>

 ● Carousel:
   A carousel component for sliding content
   Docs: <doc-url>
```

<Callout title="Tip">

Use `list` before adding components to see what’s available.

</Callout>

## Behavior

- **Add:** Logs installed components, warns for missing/skipped ones, and logs required dependencies with version info.
- **Remove:** Logs removed/failed components and logs no-longer-required dependencies (only supports manual uninstall for safety).
- **List:** Display available components with descriptions (or `(no description)` if missing).

---

## Configuration

- Config: `suic.config.json` at project root.
- Default path: `<project-root>/src/suic` (recommended to keep default).
- Custom path: `--install-path` (`-i`) during `init`.

**Example config:**

```json
{
  "installPath": "/src/suic"
}
```

## Tips

- Always run `init` first.
- Don’t manually edit/remove `suic.config.json` or `.suic/installed-registry`.
- no components specified in `add`/`remove` will list components for selection.
- Names are case-insensitive.
- Can see list with `suic-cli list`.
- CLI logs success, fail, skip for clarity.
- Refer to the docs site for latest components: [https://suic-docs.vercel.app/docs/components](https://suic-docs.vercel.app/docs/components)
